# 로컬 LLM 실행 생태계

> 연관 문서: [LLM 구동의 주요 스펙: 메모리 용량과 대역폭](./memory-bandwidth.md)

본문 §5에서 "외부망이라면 경량 모델을 로컬에 올려 쓰는 선택지가 생긴다"고 언급한 대목에서 구체적으로 어떤 도구를 어떻게 조합하는지가 이 부록입니다. 모델 저장소(HuggingFace) → 실행 엔진(Ollama / LM Studio) → 활용 레이어(OpenClaw 같은 에이전트) 세 계층이 어떻게 연결되는지, 각 도구가 어떤 문제를 푸는지를 짧게 정리합니다.

---

## 왜 로컬 실행인가

클라우드 LLM(ChatGPT, Claude API 등)은 편리하지만 세 가지 제약이 있습니다.

- **프라이버시**: 입력한 내용이 외부 서버로 전송됩니다.
- **비용**: 토큰 단위 과금이므로 대량 사용 시 비용이 빠르게 쌓입니다.
- **인터넷 의존**: 오프라인 또는 폐쇄망 환경에서 사용이 불가능합니다.

로컬 실행은 이 세 가지를 모두 해결합니다. 오픈소스 LLM을 자신의 PC나 서버에 올려두면 데이터가 외부로 나가지 않고, 사용량에 관계없이 추가 비용이 없으며, 인터넷 없이도 동작합니다.

---

## 생태계 전체 구조

로컬 LLM 실행에는 세 계층의 도구가 관여합니다.

```
+---------------------+
| HuggingFace Hub     |  모델 저장소 -- 여기서 모델을 내려받는다
+---------------------+
           |
           | 모델 파일 다운로드
           v
+----------+  +-------------+
|  Ollama  |  |  LM Studio  |  실행 엔진 -- 모델을 띄우는 도구
+----------+  +-------------+
           |
           | OpenAI 호환 로컬 API
           v
+----------+  +-----------------+
| OpenClaw |  | 개발자 앱/SDK   |  활용 레이어 -- LLM을 실제로 쓰는 곳
+----------+  +-----------------+
```

모델 파일(수십~수백 GB)은 HuggingFace Hub에서 내려받고, Ollama 또는 LM Studio가 이를 메모리에 올려 API로 노출하며, OpenClaw나 자체 개발 앱이 그 API를 통해 LLM을 활용합니다.

---

## HuggingFace Hub

> "AI 모델의 GitHub"

### 한 줄 설명

전 세계 연구자·기업이 학습시킨 AI 모델을 올려두는 공개 저장소입니다. Ollama·LM Studio가 모델을 내려받는 원천이 됩니다.

### 무엇을 할 수 있나

| 기능 | 설명 | 코딩 필요 여부 |
|---|---|---|
| **모델 검색** | Llama, Qwen, Gemma 등 200만+ 모델 검색·다운로드 | 불필요 |
| **Spaces** | 브라우저에서 바로 AI 앱 체험 (설치 없음) | 불필요 |
| **Inference API** | 클라우드에서 모델 API 호출 (월 100K 크레딧 무료) | 최소 |
| **Chat UI** | ChatGPT처럼 대화 (무료) | 불필요 |
| **AutoTrain** | UI 기반 모델 파인튜닝 | 불필요 |

### 비용

모델 다운로드와 Spaces 사용은 무료입니다. 전용 클라우드 추론 서버(Inference Endpoints)나 엔터프라이즈 기능은 유료입니다.

### 로컬 실행과의 관계

HuggingFace Hub 자체는 **클라우드 서비스**입니다. 로컬 실행을 원한다면, 허브에서 모델 파일(`.gguf`, `.safetensors` 형식)을 내려받아 Ollama나 LM Studio에 불러옵니다. LM Studio는 앱 안에서 HuggingFace 검색·다운로드를 통합 제공합니다.

## Ollama

> "터미널 한 줄로 LLM을 실행하는 도구"

### 한 줄 설명

명령줄(터미널)에서 `ollama run qwen3:14b`를 입력하면 모델을 자동으로 내려받아 즉시 대화할 수 있습니다. 개발자가 다른 앱과 연동할 때 가장 널리 사용하는 로컬 실행 엔진입니다.

### 주요 특징

**OpenAI 호환 API**: Ollama는 `localhost:11434/v1/` 주소로 REST API를 노출합니다. ChatGPT를 연결하도록 만들어진 기존 앱·라이브러리를 설정만 바꿔 로컬 모델로 전환할 수 있습니다.

**모델 라이브러리**: `ollama.com/library`에서 Llama·Qwen·Gemma·DeepSeek 등 주요 모델을 검색하고 `ollama pull <모델명>`으로 내려받습니다. 태그 페이지(`ollama.com/library/<모델명>/tags`)에는 양자화별 컨텍스트 길이·용량이 한눈에 정리되어 있어, 구매 전 하드웨어 적합도 판단에 유용합니다.

**멀티모달**: LLaVA 계열 모델로 이미지 입력을 지원합니다.

### 지원 플랫폼

macOS·Linux·Windows 모두 지원합니다.

### 최근 주요 업데이트

- **v0.21.1** (2026-04): `kimi-k2.6:cloud` 클라우드 모델 추가 (`ollama launch kimi`)
- **v0.21.0** (2026-04-16): Gemma 4 on MLX 지원, Hermes Agent, GitHub Copilot CLI 통합 (Kimi/GLM/MiniMax 신규 추가는 없으며 OpenClaw 호환은 v0.20.5부터 이미 제공됨)
- **v0.6.2** (2025-03-18): Gemma 3 멀티 이미지 입력, AMD Strix Halo 지원 (Llama 4는 v0.7.0부터)

### 비용

완전 무료·오픈소스입니다.

## LM Studio

> "ChatGPT를 내 컴퓨터에 설치한 것"

### 한 줄 설명

그래픽 UI를 갖춘 데스크탑 앱입니다. 터미널 없이 클릭만으로 모델을 검색·다운로드하고, ChatGPT처럼 대화하거나 PDF를 분석할 수 있습니다.

### 주요 특징

**모델 검색 UI**: 앱 안에서 HuggingFace 모델을 직접 검색·다운로드합니다. 권장 스펙을 미리 표시해줘서 내 PC에 맞는 모델을 고를 수 있습니다.

**로컬 API 서버**: Ollama와 마찬가지로 OpenAI 호환 API(`localhost:1234/v1/`)를 노출합니다. Python·JavaScript SDK도 제공합니다.

**LM Link** (v0.4.5+): 원격 PC의 LM Studio에 Tailscale 암호화로 연결해, 성능 좋은 데스크탑의 모델을 다른 기기에서 쓸 수 있습니다.

**MCP 서버 연결** (v0.4.10+): 외부 도구를 연결하는 MCP(Model Context Protocol)를 지원해 에이전트 기능을 확장할 수 있습니다. v0.4.10에서 MCP 서버 인증에 OAuth가 추가됐습니다.

### 지원 플랫폼

macOS·Windows·Linux(ARM 포함) 지원합니다.

### 최근 주요 업데이트

- **v0.4.12** (2026-04-17): Qwen 3.6 지원
- **v0.4.11** (2026-04): Gemma 4 채팅 템플릿 업데이트
- **v0.4.10** (2026-03): MCP 서버 OAuth 인증 지원
- **v0.4.5** (2026-02-25): LM Link — Tailscale 기반 원격 LM Studio 접속

### 비용

완전 무료입니다. (개인·업무 모두)

## OpenClaw

> "LLM이 스스로 일을 처리하는 자율 AI 에이전트"

### 한 줄 설명

단순 채팅봇이 아니라 **실제 작업을 수행**하는 에이전트 프레임워크입니다. 텔레그램·Discord를 UI로 사용하고, LLM이 이메일 읽기·웹 검색·API 호출·코드 실행 같은 100개 이상의 내장 스킬을 조합해 멀티스텝 태스크를 자율 처리합니다.

### 작동 방식

```
[사용자] --> Telegram 메시지
                  |
            [OpenClaw]
                  |
          +-------+-------+
          |               |
     [LLM 판단]      [스킬 실행]
     (Ollama 등)    (이메일, 검색,
                     쉘 커맨드 등)
```

### Claude Code와의 차이

| | Claude Code | OpenClaw |
|---|---|---|
| **특화 영역** | 소프트웨어 개발 | 범용 업무 자동화 |
| **UI** | 터미널(CLI) | 텔레그램·Discord |
| **LLM** | Claude(클라우드) | 모든 LLM (로컬 포함) |
| **사용 대상** | 개발자 | 비개발자 포함 |

> Claude Code가 "파일·쉘·에디터를 기본 도구로 갖는 개발 특화 에이전트" 라면, OpenClaw는 "메시징 플랫폼을 UI로 쓰는 범용 업무 에이전트" 입니다. 두 도구의 설계 철학·하네스 구성은 [Claude Code의 3축](../tools/claude-code-3-axes.md)에서 좀 더 구체적으로 비교합니다.

### 이름 변천

Clawdbot(2025-11) → Moltbot(2026-01-27) → **OpenClaw**(2026-01-30~현재)

Anthropic 상표 이슈로 두 차례 이름이 바뀌었습니다.

### 주요 사항

- Ollama v0.20.5부터 공식 호환 (v0.21.0은 단순 버그픽스, 신규 통합 아님)
- GitHub 저장소: [github.com/openclaw/openclaw](https://github.com/openclaw/openclaw) (org)
- 2026-02: 개발자가 OpenAI 합류, 비영리 재단으로 프로젝트 이관 예정
- GitHub 스타 시계열: 145K(2026-02) → 247K(2026-03) → 280K+(2026-03 후반) → 337K+ (2026-04 기준)
- **보안 주의**: 이메일·API·파일 시스템 접근 권한을 가지므로 설정 오류 시 데이터 유출 위험이 있습니다. 프롬프트 인젝션 공격에도 취약할 수 있습니다.

## 상황별 선택 가이드

**모델을 일단 웹에서 빠르게 체험하고 싶다**
→ HuggingFace Spaces — 설치 없이 브라우저에서 바로 사용

**내 PC에서 조용히 돌리고 싶다, 터미널은 괜찮다**
→ Ollama — 설치 한 줄, 다른 앱과 연동이 쉬움

**코딩 없이 ChatGPT처럼 쓰고 싶다**
→ LM Studio — GUI 클릭만으로 모델 설치·대화

**이메일·일정·검색 같은 반복 업무를 AI에게 맡기고 싶다**
→ OpenClaw — Ollama와 조합해 완전 로컬·자율 에이전트 구성 가능
